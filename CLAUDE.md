## 11. TOKEN & CREDIT EFFICIENCY — Auto-Apply Every Session

> Output tokens cost **5× more** than input tokens. Every optimisation focuses there first.

### 11.1 The Top 5 Rules (Biggest ROI, Implement Now)

**RULE 1 — One-Shot Prompts (saves ~60% tokens)**
Never clarify over 5 back-and-forth turns. Front-load ALL context in message 1.
Wrong: "Write a cold email" → "Make it for Malaysia" → "Add a CTA" (3× context reload = 4,000+ tokens)
Right: Role + Context + Audience + Goal + Tone + Format in one message (800 tokens, one pass).
→ JARVIS applies this automatically: if a request is under-specified, JARVIS asks one clarifying question then executes fully.

**RULE 2 — Format Constraints Upfront (saves ~40% output tokens)**
Always specify length and format BEFORE Claude writes anything.
Add to any prompt: "Max 100 words · 3 bullets · Subject line under 50 chars · JSON format"
Claude with no constraints → 800 tokens. Claude with constraints → 250 tokens. Same quality.

**RULE 3 — Model Routing (saves up to 19× cost)**
- **Haiku 4.5** — simple copy, templates, classifications, summaries (1× cost)
- **Sonnet 4.6** — DEFAULT for all daily work: code, strategy, analysis, writing (3× cost)
- **Opus 4.6** — ONLY when Sonnet genuinely fails: novel architecture, complex multi-file reasoning (19× cost)
Never default to Opus. 90% of tasks Sonnet handles at equivalent quality.

**RULE 4 — Batch Related Questions (saves ~30% tokens)**
Don't send 4 separate messages. Group all related questions in one prompt with numbered items.
JARVIS applies this automatically: if a multi-part task arrives across messages, JARVIS consolidates before executing.

**RULE 5 — Trim Files Before Review (saves ~70% on file tasks)**
Never paste a 400-line file and ask "what do you think?"
Extract the 20 lines that matter. Reference the rest by filename.
JARVIS applies this automatically: reads only the relevant section of any file before passing to analysis.
